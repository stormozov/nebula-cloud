# Nebula Cloud

[🇬🇧 EN version](./README.md)

Fullstack веб-приложение облачного хранилища файлов.

Веб-приложение для хранения, управления и обмена файлами. Пользователи могут загружать, скачивать, переименовывать, удалять файлы, а также делиться ими с помощью специальных публичных ссылок. Интерфейс представляет собой одностраничное приложение (SPA) на React + TypeScript, серверная часть — REST API на Django, данные хранятся в PostgreSQL.

## Основные возможности

### Аутентификация и управление пользователями

* Регистрация с валидацией:
  - Логин – только латиница и цифры, первый символ буква, длина 4–20 символов.
  - Email – проверка на соответствие стандартному формату.
  - Пароль – минимум 6 символов, обязательно присутствие заглавной буквы, цифры и спецсимвола.
* Вход по логину и паролю, отслеживание сессии и обновление токена.
* Администратор может:
  - просматривать список всех пользователей,
  - просматривать данные о конкретном пользователе,
  - изменять данные и пароль конкретного пользователя,
  - изменять признак администратора и статус активности аккаунта,
  - удалять пользователей.
* В списке пользователей отображаются данные хранилища:
  - количество файлов,
  - общий размер,
  - ссылка для перехода к управлению файлами.
* При клике на выбранного в списке пользователя открывается модальное окно на весь экран с информацией о пользователе и кнопками управления.
* Экспорт информации о пользователе в формате JSON.

### Работа с файловым хранилищем

* Загрузка файлов через кнопку и дроп-зону.
* Просмотр загруженных файлов в панели загрузки файлов с указанием состояния загрузки и с возможностью отменить загрузку или повторить при ошибке.
* Просмотр списка файлов с информацией: оригинальное имя, комментарий, размер, дата загрузки, дата последнего скачивания.
* Операции с файлами: скачивание, переименование, удаление, копирование специальной ссылки. Операции могут быть вызваны в виде контекстного меню при клике на кнопку у каждого пункта списка, либо нажатии на ПКМ.
* Поиск по имени, дате загрузки.
* Специальная публичная ссылка – обезличенный URL, по которому файл можно скачать без аутентификации; при скачивании сервер отдаёт файл с оригинальным именем.
* Обычные пользователи управляют только своими файлами; администратор может управлять хранилищем любого пользователя.
* Модальное окно со справкой о горячих клавишах, используемых в файловом менеджере.
* Прогресс-бар, показывающий занятый объем диска.
* Пагинация списка файлов с подгрузкой с помощью кнопки "Загрузить ещё". 

### Интерфейс

* Реализован по концепции SPA: все динамические данные загружаются через API.
* Навигационное меню изменяется в зависимости от состояния аутентификации (Вход, Регистрация, Выход).
* Интуитивно понятный интерфейс, ориентированный на пользователей, знакомых с облачными хранилищами.
* Адаптивность под разрешения мониторов ПК.
* Возможность смены темы интерфейса (светлая, темная и системная).
* Поддержка спиннеров и скелетонов во время ожидания ответа сервера.
* Поддержка уведомлений при взаимодействии с интерфейсом.
* Поддержка контекстного меню для файлового менеджера и списка пользователей в панели администратора.
* Поддержка управления интерфейсом через клавиатуру.
* Поддержка анимаций при взаимодействии с интерфейсом.
* Приветливая стартовая страница и страница 404.

## Технологический стек
* Backend: Python, Django, Django REST Framework, JWT, PostgreSQL
* Frontend: TypeScript, React, Redux Toolkit, React Router, Vite
* Хранение файлов: файловая система сервера
